User Story - INVEST
课后练习题
________________
简答题
1. 用自己的话简述INVEST原则每个字母的含义。(20分)
  I:Independent;意味着开发的模块是相对独立的，开发当前模块不需要有其他先决条件就可以进行开发。
  N:Negotiable;意味着当前的用户故事并不是一成不变的，如果发现这样开发有什么浪费时间资源的问题或者逻辑上的问 题，是可以多方进行协商，直到完成出大家都认可的。
  V:Valuable;意味着在开发时不要只面对功能，要想实现改功能的目的，能给需求改功能的人员提供哪些价值，这些价值是不是他们需要的，还可不可以再改进。
  E:Estimable;当拿到用户故事时，可以估算出想要实现该功能需要多长时间，可以确保每一个故事所用的时间都是能计算出正确的合理的需要的时间点。
  S:Small;也就是用户故事是小的切分，目标很明确，需求很明了，实现时也知道具体怎么做。
  T:Testable;对于判定当前用户故事是否完成是有具体的考核标准的，不是模糊不清的，而且这种标准是大众所认可，不因人而异的。

2. 用自己的话简述用户故事两种常见的切分方式。(20分)
  垂直切分：垂直切分是当前模块的前后端是一个整体，在切分时包含UI层，业务逻辑层以及数据存储层，即当完成之后一个功能的前后端就都有了。
  水平切分：前端和后端是分开的，前端人员关注前端的功能实现，后端人员关注后端的功能实现，前后端人员在定义了一些数据的传输，定义方式之后就进行独立开发。


情景题
根据以下用户故事概述，判断该用户故事是不是符合INVEST原则？如果不符合，说明违背了哪个或哪几个原则，并解释原因。


<Case 1> (20分)
As a customer,
I want to be able to shop online
So that I can do it wherever and whenever I want, without physically going to the store

不符合规则：违背了E，S，T。实现购物的需求不明确，无法估算需要多长时间，是小程序式的线上购物还是网页版的线上购物，用户提供什么信息能进行购物，这些都是不明确的，即无法估算完成时间又没有一个明确的衡量任务完成的标准，太笼统。


<Case 2>(20分)
As a customer
I want to update and delete items in my shopping cart
So that I can change my mind about what to buy

不符合。违背了I。虽然用户的明确需求，就是对购物车进行增加或者删除，也可以估计出所需时间以及进行判断是否完成时也可以通过测试是否可以增加删除商品来确定是否完成了，但是想要增加新的商品是需要一个商品选择模块的，在商品选择模块完成后才能完成购物车的操作，应该细分为加购商品模块和删除购物车商品模块。


<Case 3>(20分)
As a customer
I want to search for the items that I needed to buy
So that I can select the items I need to buy

符合。因为很明确用户需要做什么，就是设计一个搜索页面，后端有商品的数据，通过对数据库的查找来找出对应商品。
